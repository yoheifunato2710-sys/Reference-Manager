# 文献管理（Paper Manager）

論文・文献をフォルダ単位で管理する**デスクトップアプリ**です。  
OneDrive などのフォルダにデータを保存し、どの端末から開いても同じデータを参照できます。

---

## デスクトップアプリとして使う（推奨）

### 1. インストーラーを作成する（初回のみ）

```bash
npm install
npm run electron:build
```

→ **「文献管理 Setup 1.0.0.exe」** が次の場所にできます。

| 場所 |
|------|
| **`C:\Users\あなたのユーザー名\AppData\Local\文献管理-installer\文献管理 Setup 1.0.0.exe`** |

※ プロジェクトが OneDrive 内にあると「Access is denied」で失敗することがあるため、出力先を上記（OneDrive 外）にしています。  
※ プロジェクト直下の `release/` に出したい場合は、実行前に `set ELECTRON_BUILDER_OUTPUT=release` を指定してください。

### 2. インストール

- 上記フォルダの **「文献管理 Setup 1.0.0.exe」** をダブルクリックして実行
- 画面の指示に従ってインストール（インストール先を変更できます）
- インストール時に **デスクトップにショートカット** が自動作成されます

### 3. 起動

- **デスクトップの「文献管理」** をダブルクリック、またはスタートメニューから「文献管理」を起動
- 初回は「フォルダを選択して開く」で OneDrive 内のフォルダを選ぶと、その中に `paper-manager-data.json` と `pdfs/` が保存されます

### 開発中にデスクトップアプリで試す

```bash
npm run start
```
（`npm run build` + `electron .` と同じ。ビルド後にウィンドウが開きます）

開発時のみホットリロードで試したい場合:
- ターミナル1: `npm run dev`
- ターミナル2: `npm run electron:dev`

### フォルダをダブルクリックで開く

- データフォルダを選んだあと、サイドバー上部の「**このフォルダにショートカット（ダブルクリックで起動）**」を押すと、そのフォルダ内に「文献管理を開く」ショートカットが作成されます
- OneDrive で同期しているフォルダにこのショートカットを置けば、そのショートカットのダブルクリックでアプリが起動し、同じデータを開けます

### アプリを最新版にアップデートする

コードを更新したあと、インストール済みの「文献管理」を最新版にする手順です。

1. **プロジェクトを最新にする**  
   - 例: `git pull` でリポジトリを更新、または自分で編集した内容を保存

2. **バージョン番号を上げる（任意）**  
   - `package.json` の `"version": "1.0.0"` を `"1.0.1"` のように変更すると、インストーラー名が「文献管理 Setup 1.0.1.exe」になります（省略しても問題なく上書きインストールできます）

3. **新しいインストーラーを作る**
   ```bash
   npm run electron:build
   ```

4. **新しいインストーラーを実行する**  
   - `%LOCALAPPDATA%\文献管理-installer\文献管理 Setup x.x.x.exe` をダブルクリック  
   - 既に「文献管理」が入っている場合は**上書きインストール**され、デスクトップのショートカットやデータフォルダの設定はそのまま使えます

※ 自動アップデート（起動時に更新を確認する機能）は入っていません。更新のたびに上記の手順で新しいインストーラーを実行してください。

## スタンドアロンで使う（ブラウザ・どの端末からでも）

1. **ビルド**
   ```bash
   npm install
   npm run build
   ```
2. **OneDrive 内のフォルダに配置**
   - `dist` フォルダの中身（`index.html` と `assets/`）を、OneDrive の「論文管理」用フォルダにコピーする  
   - 例: `OneDrive/開発/論文管理/` に `index.html` と `assets/` を置く

3. **開き方**
   - **PC**: そのフォルダの `index.html` をブラウザで開く（ダブルクリック、またはブラウザで「ファイルを開く」）
   - **別の端末**: OneDrive 同期で同じフォルダが共有されていれば、その端末でも同じフォルダ内の `index.html` を開く
   - 初回のみ「フォルダを選択して開く」で、その **同じフォルダ**（データを保存したいフォルダ）を選ぶ

4. **データの保存場所**
   - 選択したフォルダに次のファイルが作成・更新されます：
     - `paper-manager-data.json` … 論文一覧・フォルダ構成
     - `pdfs/` … 添付 PDF（論文ごとに `{id}.pdf`）

編集すると自動で保存され、同じフォルダを開いている他の端末では OneDrive の同期後に最新データが反映されます。

## 開発モード（AI 抽出を使う場合）

```bash
npm run dev
```

- ブラウザで表示された URL（例: http://localhost:5173）を開く
- `.env` に `ANTHROPIC_API_KEY` または `GEMINI_API_KEY` を設定すると、論文追加時の「AIで抽出」が使えます

## フォルダ選択が使えないブラウザの場合

- **データを読み込み**: 「データファイル（JSON）を読み込む」で、以前エクスポートした `paper-manager-data.json` を選ぶ
- **保存**: サイドバーの「データをエクスポート（JSON）」で JSON をダウンロードし、同じ OneDrive フォルダに置いておけば、他の端末で読み込めます

## 技術メモ

- フォルダの読み書きには **File System Access API**（Chrome / Edge など）を使用しています。Safari などでは上記の「読み込み・エクスポート」で運用してください。
- スタンドアロンで開くときは `index.html` を **ファイルで開く**（`file://`）か、同じフォルダをドキュメントルートにして簡易 HTTP サーバーで配信してください。
